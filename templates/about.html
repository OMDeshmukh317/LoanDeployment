About the Project

This project is a machine learning-based classification system designed to predict loan status (approved or rejected) based on various borrower characteristics. By analyzing key factors such as income, credit history, and loan amount, the project provides insights into loan eligibility and automates the decision-making process for financial institutions. The goal is to develop a robust and efficient model that handles diverse data types while ensuring high predictive accuracy.

Project Objectives

  1. Explore and preprocess a real-world dataset containing demographic, financial, and credit-related features.
  2. Build and evaluate multiple machine learning models for classification.
  3. Optimize the best-performing model using advanced hyperparameter tuning techniques.
  4. Deploy the final model into a user-friendly web application for easy access and interaction.

Technologies Used

Dataset Overview

The dataset used for this project consists of 45,000 entries and 14 features, sourced from Kaggle. It includes a mix of numerical and categorical variables, representing diverse borrower characteristics such as:

  • Demographics: Age, gender, and education level.
  • Financial Details: Income, employment experience, and home ownership status.
  • Loan Features: Loan amount, interest rate, loan intent, and percentage of income dedicated to loans.
  • Credit History: Credit score, credit history length, and previous loan defaults.

The target variable, loan_status, indicates whether the loan was approved (1) or rejected (0).

Key Steps in Model Development

  1. Data Preprocessing: The dataset was pre-cleaned, eliminating the need for additional handling of missing values.
  2. Outlier Detection: Identified and addressed to enhance model reliability.
  3. Data Visualization: Explored feature distributions and relationships using EDA techniques.
  4. Feature Encoding: Transformed categorical variables using Label Encoding for compatibility.
  5. Class Imbalance Handling: Used SMOTE to address target variable imbalance.
  6. Model Training: Evaluated the following algorithms using cross-validation:
    • Logistic Regression
    • Decision Tree Classifier
    • Random Forest Classifier
    • Gaussian Naive Bayes
    • K-Nearest Neighbors (KNN)
    • AdaBoost
    • XGBoost
  7. Hyperparameter Tuning: Optimized models using GridSearchCV.

Results and Performance

The XGBoost model emerged as the best performer, achieving an accuracy of 93%. This demonstrates the effectiveness of preprocessing and model selection.

Deployment

The final model was deployed using Django, creating a user-friendly web application. The system is hosted on Render, making it accessible for real-world use.

Future Enhancements

Potential improvements for the project include:

  • Incorporating additional features, such as geographical data, to improve prediction accuracy.
  • Developing an API for integration with banking platforms.
  • Optimizing deployment for improved response times and scalability.
  • Adding advanced visualization tools for real-time insights.
